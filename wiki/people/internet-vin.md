# Internet Vin

**Toronto-based builder whose public workflow for using Obsidian as a thinking-partner second brain — paired with Claude Code via the Obsidian CLI — is the second major influence on this vault, after [[andrej-karpathy]]. Source of the [[vindates]].**

## Summary
On Greg Eisenberg's podcast (from [[2026-05-19--internet-vin-obsidian-claude]]) Vin demonstrated a dense custom slash-command set built on top of Obsidian + Claude Code. His framing differs from [[andrej-karpathy]] on one key point: **the LLM should not write into the vault**. He keeps a strict separation between his own notes and the agent's output, so that pattern-detection across the vault stays a mirror of his own thinking, not a feedback loop on the agent's prior writing.

## Key points
- Heavy user of Obsidian's local-graph and inter-link features — the graph, not the folder, is the artifact.
- Built ~12 custom slash commands for daily ops and reflection (see [[slash-commands]]).
- Strong philosophical commitment to **LLM-as-thinking-partner over LLM-as-builder**.
- Treats the vault as the source of context to delegate from — "I manage the vault, not the agent."
- Acknowledges the privacy weight of giving an autonomous agent broad vault access.
